Projet unifie

ElephantNote

Un seul projet organise autour d'un coeur C modulaire, d'un moteur Markdown, d'une gestion de curseur dediee, d'un build WASM reproductible et d'une interface web publiee proprement sur GitHub Pages.

Composants

Chaque brique a une responsabilite nette.

Core

Markdown Engine

Parsing Markdown, representation JSON intermediaire, export Markdown et normalisation des structures du document.

Edition

Editor ABI

API stable pour les integrations, gestion d'etat, conversion Markdown/JSON/HTML et surface WASM publique.

Interaction

Cursor Engine

Positionnement intelligent, Enter/backspace coherents, preservation du formatage et edition hybride ligne par ligne.

Web

GitHub Pages

Landing unique, demo principale, outils de verification et artefacts generes depuis une seule chaine de build.

Native

macOS / Flutter

Interfaces clientes autour du meme coeur C, sans dupliquer la logique metier de parsing ou de curseur.

Qualite

Tests et Validation

Tests unitaires moteurs, verification WASM, et page publique preparee depuis des artefacts valides.

Navigation

Les entrees utiles uniquement, sans liens morts.

Demo

Editeur principal

Interface hybride branchée sur les modules `editor` et `cursor` en WebAssembly.
Verification

Test live Enter

Validation interactive du comportement de la touche Entree dans les cas de formatage delicats.
Regression

Correctif curseur

Page ciblee pour rejouer les scenarios qui ont pose probleme pendant le developpement.
Build GitHub Pages : la publication ne depend plus d'un ancien dossier `docs/` a la racine. Le workflow construit maintenant les artefacts depuis la racine du depot, copie les bundles WASM dans `web/site/docs` puis publie l'ensemble de `web/site`.
